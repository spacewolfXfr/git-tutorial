# Teaching ideas

- Hide branch complexity in the beginning ("just do these commands, we will explain them later")
- OR use a different setup to simplify this (but may require 'local' remote or other shenanigans...)
- Create slideshow for the tutorial, containing screenshots of a teacher command line, to show the commands and the effect you are supposed to get
- Refrain students to move forward too fast (and so, hide the 'teaching_material' from them). It makes sure the push and pull create the expected conflicts
